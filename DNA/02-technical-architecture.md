# GnoccaTrack — Technical Architecture

## Stack

| Layer | Technology |
|---|---|
| Frontend | React 19 + Vite 7 |
| Styling | Tailwind CSS v4 + tw-animate-css |
| UI Components | Radix UI (shadcn/ui conventions) |
| Routing | Wouter |
| State / Data | TanStack React Query v5 |
| API client | Generated by Orval from OpenAPI spec |
| Animations | Framer Motion |
| Map | MapLibre GL JS |
| Map tiles | OpenFreeMap (free, open) |
| Venue detection | Overpass API (OpenStreetMap) |
| Backend | Express 5 |
| Database | PostgreSQL + Drizzle ORM |
| Validation | Zod (zod/v4) + drizzle-zod |
| API codegen | Orval (OpenAPI → React Query hooks + Zod schemas) |
| Build | esbuild (server), Vite (client) |
| Workspace | pnpm monorepo |

## Monorepo Structure

```
artifacts/
  api-server/       # Express 5 backend
  gnocca-track/     # React + Vite PWA frontend
lib/
  api-spec/         # OpenAPI spec (source of truth)
  api-client-react/ # Generated React Query hooks
  api-zod/          # Generated Zod schemas for server
  db/               # Drizzle schema + db connection
DNA/                # Project documentation
```

## Data Flow

```
Browser GPS → Overpass API (venue detection) → user taps "Qui Gnocca"
→ POST /api/clicks (Express) → upsert hotspot + record click → 200
→ React Query invalidates hotspots → map re-renders markers
```

## Service Routing

A shared reverse proxy routes by path:
- `/api/*` → Express API server (port from env)
- `/*` → Vite frontend (port from env)

## API Contract

OpenAPI spec at `lib/api-spec/openapi.yaml` is the single source of truth.
After any spec change, run: `pnpm --filter @workspace/api-spec run codegen`

## Authentication

No session-based auth in baseline. User identity is established by:
1. `deviceId` — UUID stored in `localStorage` key `gt_device_id`
2. `nickname` — chosen during onboarding
3. `pinHash` — scrypt hash with per-user random salt, stored server-side

The device ID is used for click rate limiting and abuse prevention.
